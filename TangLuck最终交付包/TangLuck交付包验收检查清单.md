# Tang Luck 交付包验收检查清单

## 1. 文档验收

| 检查项 | 通过标准 | 结果 |
| --- | --- | --- |
| 主 PRD 可追溯 | 交付包能承接主 PRD 的范围、阶段、合规边界 | 待评审填写 |
| 文件索引清晰 | 每份文档有角色、用途、阅读顺序 | 待评审填写 |
| 原型可打开 | `TangLuck高保真原型.html` 可在浏览器打开，无乱码 | 待评审填写 |
| P0-A 范围明确 | 15 天冲刺表能说明每天做什么、验收什么 | 待评审填写 |
| 活动配置可执行 | P0-A 活动配置参数表能说明奖励、预算、SC 策略和发布阻断 | 待评审填写 |
| 演示脚本可执行 | 15 天客户演示脚本能串起 C 端、后台、风控、审计 | 待评审填写 |
| P0-B/P1/P2 有路线 | 15 天后路线能说明后续先后顺序和阻断条件 | 待评审填写 |

## 2. 产品与运营验收

| 模块 | 检查项 | 通过标准 |
| --- | --- | --- |
| 注册 | 年龄、地区、条款确认 | 未成年、禁止州、未同意条款不能进入发奖链路 |
| 首页 | 钱包、奖励、任务、Coupon、规则入口 | 用户能理解当前可做什么、资产是多少、规则在哪里 |
| 钱包 | GC/SC、冻结 SC、ledger、SC 来源说明 | GC 和 SC 不混同，所有资产变化可追溯 |
| 活动 | 注册赠送、每日登录、每日任务、Coupon | 每个活动有奖励、预算、地区、SC 策略 |
| 商店 | Gold Coins Pack | 只表达购买 GC，不出现购买 SC |
| 兑换 | 兑换资格、KYC、冻结、审核状态 | 未满足条件时明确提示，不承诺保证到账 |
| AMOE | No Purchase Necessary 免费路径 | 用户能找到 AMOE 入口和规则说明 |
| 客服 | 工单入口、FAQ、业务单关联 | 客服不能绕过规则直接补发 SC |

## 3. 后台与研发验收

| 模块 | 检查项 | 通过标准 |
| --- | --- | --- |
| 权限 | 后台 RBAC | 运营、客服、风控、法务、管理员权限边界清晰 |
| 活动配置 | 类型、地区、奖励、预算、规则版本 | 缺少必填项不能发布 |
| SC 策略 | `gc_only`、`default_small_sc`、`legal_required`、`sc_blocked` | 后端强校验，不只依赖前端 |
| 钱包账本 | `wallet_accounts` + `wallet_ledger` | 发奖、冻结、解冻、扣减都有流水 |
| 幂等 | 发奖、支付回调、兑换冻结、客服补偿 | 重复请求不重复入账 |
| 审计 | 后台写操作 | 记录操作人、对象、前值、后值、时间、IP |
| 风控 | IP、设备、多账号、地区 | 风险用户只发 GC、延迟、人工审核或拦截 |
| 看板 | 注册、领取、SC、风控、兑换 | 能支持 15 天演示和运营复盘 |

## 4. 合规验收

| 检查项 | 通过标准 |
| --- | --- |
| Terms / Rules / Privacy | 注册页、页脚、活动页可访问并版本化 |
| No Purchase Necessary | 独立入口，不只藏在规则长文 |
| AMOE | 有说明页，P0-B/P1 有申请和审核路径 |
| SC 不销售 | 商店、订单、收据、按钮不表达为购买 SC |
| 购买不提高中奖机会 | 购买 GC 不作为获得 SC 或 prize 的唯一方式 |
| 州级控制 | 至少支持 `state_code` 级开关 |
| 法务审批字段 | SC、AMOE、转盘、排行榜高价值奖品、SC bonus 有 `legal_approval_id` |
| 客服话术 | 不承诺保证兑换、保证到账、绕过审核 |

## 5. QA 必测路径

| 路径 | 验收场景 |
| --- | --- |
| 正常用户注册领奖 | 注册、条款、钱包创建、注册奖励、ledger |
| 禁止州用户 | 注册或功能被限制，后端返回 `REGION_BLOCKED` |
| 风险用户领奖 | 风险命中后只发 GC 或进入人工审核 |
| 重复领取 | 重复请求返回 `CLAIM_DUPLICATED`，不重复入账 |
| Coupon 领取 | 有效、过期、已用、批次限制、撞库限制 |
| 活动发布 | 缺地区、预算、规则版本、法务审批时阻断 |
| 钱包流水 | 每笔 GC/SC 变化可查业务来源和幂等 key |
| AMOE | 静态入口可访问，P0-B 后申请可审核、可发奖 |
| 兑换 | SC 不足、KYC 未通过、冻结、审核、拒绝 |
| 后台审计 | 每次发布、审核、补偿都写审计 |

## 6.1 15 天客户演示验收

| 演示项 | 通过标准 |
| --- | --- |
| 正常用户主链路 | 注册、领取注册赠送、每日登录、任务、Coupon、钱包流水连续可演示 |
| 风险用户链路 | 风险用户领取时只发 GC 或被拦截 |
| 禁止州链路 | 禁止州用户触发地区限制 |
| 后台活动配置 | 可查看和编辑活动配置 |
| 发布阻断 | 缺少 `legal_approval_id` 的 SC 活动不能发布 |
| 审计日志 | 后台写操作可查操作人、前后值、时间、IP |
| 合规入口 | Terms、Rules、No Purchase Necessary、AMOE 可访问 |

## 6. 上线阻断项

| 阻断项 | 处理方式 |
| --- | --- |
| 无州级合规配置 | 不允许上线真实 SC、购买、兑换 |
| 活动发 SC 无 `legal_approval_id` | 不允许发布 |
| 钱包无 ledger | 不允许发奖 |
| 后台无审计 | 不允许运营配置上线 |
| No Purchase Necessary / AMOE 不可访问 | 不允许上线 sweepstakes 模式 |
| 商品包表达为购买 SC | 不允许上线商店 |
| 支付商未准入 | 不允许真实支付 |
| KYC/打款供应商未确认 | 不允许真实兑换 |
| App 审核材料缺 Rules/AMOE/地区说明 | 不建议提审 |
