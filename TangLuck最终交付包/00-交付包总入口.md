## 1. 交付包定位

本目录用于承接主文档 `TangLuck可运营可评审可验收最终PRD.md`，把 Tang Luck 美国 Sweepstakes Casino 平台从产品方案拆成可评审、可开发、可测试、可外部确认的专项交付物。

| 场景 | 使用方式 |
| --- | --- |
| 客户评审 | 先看本文，再看高保真原型、15 天冲刺表、15 天后路线 |
| 研发开工 | 先看接口草案、数据库设计、页面原型说明、15 天冲刺表 |

| 支付商准入 | 使用支付商准入包，重点确认只卖 GC、不卖 SC、退款拒付 |
| App 提审准备 | 使用 App 上架审核包，准备审核账号、规则入口和地区说明 |

## 4. 15 天 P0-A 必须守住的边界

| 能力 | P0-A 做法 | 不建议在 P0-A 做 |
| --- | --- | --- |
| 注册 | 邮箱注册、年龄确认、条款确认、地区限制 | 复杂 KYC、手机号强绑定 |
| 钱包 | GC/SC 分账、余额、ledger、SC 来源说明 | 直接改余额、不留流水 |
| 活动 | 注册赠送、每日登录、每日任务、Coupon | 邀请、排行榜、转盘大规模上线 |
| SC 策略 | `gc_only`、`default_small_sc`、`legal_required`、`sc_blocked` | 未审批活动直接发 SC |
| 合规入口 | Terms、Rules、Privacy、No Purchase Necessary、AMOE 静态入口 | 把免费路径藏在长文里 |
| 后台 | 活动配置、钱包流水、发奖记录、审计、基础看板 | 自动风控、大规模运营自动化 |
| 购买 | 预留商品包和订单结构 | 接真实支付 |
| 兑换 | 灰态说明或 P0-B 申请预留 | 真实打款 |
| App | 准备材料 | 依赖 App 上架作为唯一上线方式 |
