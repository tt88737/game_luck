# Tang Luck QA 测试用例矩阵

## 1. P0-A 必测

| 模块 | 测试场景 |
| --- | --- |
| 注册 | 成功、重复邮箱、未勾选条款、未成年、禁止州 |
| 钱包 | GC 入账、SC 入账、流水、幂等、防重复 |
| 活动 | 正常领取、重复领取、过期、预算用尽、禁止州 |
| 任务 | 完成进度、领奖、重复上报、周期重置 |
| Coupon | 有效、过期、已用、批次限制、撞库限制 |
| SC 策略 | `gc_only`、`default_small_sc`、`legal_required`、`sc_blocked` |
| 风控 | IP、设备、多账号、风险用户只发 GC |
| 合规入口 | Terms、Rules、Privacy、No Purchase、AMOE |
| 后台 | 活动新增、编辑、发布、暂停、审计 |
| BI | 注册、领取、SC 总量、风控拦截 |

## 2. P0-B/P1 必测

| 模块 | 测试场景 |
| --- | --- |
| 兑换 | SC 不足、KYC 未通过、冻结、审核、拒绝、失败 |
| AMOE | 提交、超频、审核、拒绝、发 SC、ledger |
| 支付 | 成功、失败、超时、重复回调、退款、拒付 |
| KYC | 通过、失败、补资料、地址不一致 |
| App 上架 | 审核账号、规则页、地区限制、支付说明 |

## 3. 验收输出

| 输出物 | 标准 |
| --- | --- |
| 测试用例 | 覆盖 P0-A 全主链路 |
| 缺陷清单 | 标注 S1/S2/S3 |
| 验收报告 | 包含通过率、阻断项、遗留风险 |
| 演示账号 | 至少正常用户、风险用户、禁止州用户 |
